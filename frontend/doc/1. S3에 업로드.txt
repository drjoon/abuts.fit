정적 웹사이트 도메인 연결을 원한다면?

1. S3 버킷을 abuts.fit 이름으로 새로 만들고 정적 웹 호스팅 활성화
aws s3api create-bucket \
  --bucket abuts.fit \
  --region ap-south-1 \
  --create-bucket-configuration LocationConstraint=ap-south-1

2. 정적 웹사이트 활성화
aws s3 website s3://abuts.fit/ --index-document index.html --error-document index.html

3. 퍼블릭 액세스 차단 해제
aws s3api put-public-access-block \
  --bucket abuts.fit \
  --public-access-block-configuration 'BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false'

4. 버킷 정책(퍼블릭 읽기) 적용
aws s3api put-bucket-policy --bucket abuts.fit --policy '{
  "Version":"2012-10-17",
  "Statement":[{
    "Sid":"PublicReadGetObject",
    "Effect":"Allow",
    "Principal": "*",
    "Action":["s3:GetObject"],
    "Resource":["arn:aws:s3:::abuts.fit/*"]
  }]
}'

5. 정책 내용을 bucket-policy.json 파일로 저장
aws s3api put-bucket-policy --bucket abuts.fit --region ap-south-1 --policy file://doc/bucket-policy.json

6. S3 버킷 CORS 설정
aws s3api put-bucket-cors --bucket abuts.fit --cors-configuration file://doc/cors.json

7. S3 버킷 CORS 설정 확인
aws s3api get-bucket-cors --bucket abuts.fit

* 배포 ID 확인
aws cloudfront list-distributions

8. 빌드 후 정적 파일을 abuts.fit 버킷에 업로드
npm run build
aws s3 sync ./dist s3://abuts.fit --region ap-south-1
aws cloudfront create-invalidation \
  --distribution-id E2CEDBBNSXCSGZ \
  --paths "/*"

# CloudFront 캐시 무효화
aws cloudfront create-invalidation \
  --distribution-id E2CEDBBNSXCSGZ \
  --paths "/*"

9. Route 53에서 abuts.fit 도메인에 S3 웹사이트 엔드포인트(Alias) 연결
S3 웹사이트 엔드포인트: abuts.fit.s3-website.ap-south-1.amazonaws.com
