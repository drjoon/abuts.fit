1. 운영 프로세스 재정의
   실제 운영 기준으로 포장.발송 플로우를 이렇게 정리했습니다.

1단계
오후 2시경 오늘 발송할 우편함들의 운송장(라벨)을 먼저 출력/저장
2단계
출력한 라벨을 들고 창고에서 우편함 실물과 라벨 내용 대조
3단계
맞는 우편함만 박스 포장 후 라벨 부착
4단계
안 맞는 우편함은 다시 우편함으로 복귀
5단계
웹앱을 실제 창고 기준으로 수정
중복 제품은 하나만 남기고 제거
누락 제품은
일부만 오늘 발송하거나
해당 우편함 전체를 내일로 미루는 방식 중 선택
6단계
수정 완료 후에만 택배 접수
7단계
뒤늦은 누락/추가 발견 시 수정 -> 재출력 -> 재접수 반복
8단계
오후 4시 집하, 코드 11이면 picked_up
9단계
배송 완료 시 completed
취소는 canceled
오류는 error
분실은 별도 예외 처리
10단계
에러 시 재시도 가능
